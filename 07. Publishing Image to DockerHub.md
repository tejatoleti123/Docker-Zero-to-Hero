# 07. Publishing Image to DockerHub

In the previous section, we built our own custom Docker image.

Now we take the next important step:

Sharing that image with others using Docker Hub.

Docker Hub is a public container registry where you can:

- Store images
- Share images
- Version images
- Pull images from anywhere

# ğŸ· Step 3: Tag Your Image Properly

Before pushing, your image must include your Docker Hub username.

If your image is:

    my-flask-app:v1

Tag it like this:

    docker tag my-flask-app:v1 <your-username>/my-flask-app:v1

Example:

    docker tag my-flask-app:v1 john123/my-flask-app:v1

Verify:

    docker images

You should now see:

    john123/my-flask-app    v1

# ğŸš€ Step 4: Push Image to Docker Hub

Run:

    docker push <your-username>/my-flask-app:v1

Example:

    docker push john123/my-flask-app:v1

Docker will:

1. Check authentication
2. Upload image layers
3. Skip layers already present
4. Confirm push success

You should see:

    latest: digest: sha256:...

This means the image was successfully uploaded.

Your image is now publicly accessible.

# ğŸ§  Understanding Image Versioning

Avoid using:

    :latest

Instead use versioned tags:

    v1
    v1.0.0
    v1.1.2

Why?

- Clear version control
- Easier rollback
- Predictable deployments
- Better CI/CD workflows

In production environments, `latest` should not be used.

---

# ğŸ” Public vs Private Repositories

Docker Hub allows:

- Public repositories (free, visible to everyone)
- Private repositories (restricted access)

For learning, public repositories are fine.

For production systems, private repositories are recommended.
