# ğŸ§ Install Docker on Linux

The recommended way to install Docker on Linux is through the official Docker repository.

The steps below are for Ubuntu (most common setup).

If you use a different distribution (CentOS, RHEL, etc.), refer to Dockerâ€™s official documentation for distribution-specific commands.

---

## Step 1: Update Package Index

    sudo apt update

---

## Step 2: Install Required Packages

    sudo apt install apt-transport-https ca-certificates curl software-properties-common -y

---

## Step 3: Add Dockerâ€™s Official GPG Key

    curl -fsSL https://download.docker.com/linux/ubuntu/gpg | sudo gpg --dearmor -o /usr/share/keyrings/docker-archive-keyring.gpg

---

## Step 4: Add Docker Repository

    echo "deb [arch=$(dpkg --print-architecture) signed-by=/usr/share/keyrings/docker-archive-keyring.gpg] https://download.docker.com/linux/ubuntu $(lsb_release -cs) stable" | sudo tee /etc/apt/sources.list.d/docker.list > /dev/null

---

## Step 5: Install Docker Engine

    sudo apt update
    sudo apt install docker-ce docker-ce-cli containerd.io -y

---

## Step 6: Verify Installation

    sudo systemctl status docker

You should see:

    Active: active (running)

---

## Step 7: Run Docker Without sudo (Optional but Recommended)

By default, Docker requires sudo.

To allow your user to run Docker commands:

    sudo usermod -aG docker $USER

Then log out and log back in.

Verify:

    docker --version

If it works without sudo, setup is complete.

# ğŸ” Verify Docker Is Working Properly

Run the test container:

    docker run hello-world

What happens:

1. Docker checks if the image exists locally.
2. If not, it pulls from Docker Hub.
3. It creates a container.
4. The container prints a success message.

If you see a message saying:

    Hello from Docker!

Then your installation is successful.

# ğŸ§  Common Installation Issues

## Permission Denied

If you see:

    permission denied while trying to connect to the Docker daemon

It means your user is not in the docker group.

Run:

    sudo usermod -aG docker $USER

Then log out and log back in.

---

## Docker Daemon Not Running

If Docker is not running:

    sudo systemctl start docker

To enable Docker at boot:

    sudo systemctl enable docker
